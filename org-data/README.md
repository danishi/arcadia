# org-data/ -- 組織固有データ

ARCADIA の提案・見積スキルが参照する**提案主体（自社）の固有情報**を格納するディレクトリ。
RFP ごとに変わらない「自社の資産」をここに整備しておくことで、プロジェクト横断で一貫した提案品質を実現する。

---

## なぜ必要か

ARCADIA は RFP 文書（顧客側の情報）をプロジェクトの `docs/` に取り込む仕組みを持つが、
提案を書くにはもう一方の軸――**自社側の情報**――も不可欠である。

| 情報の軸 | 格納先 | ライフサイクル |
|---------|--------|-------------|
| 顧客・案件固有（RFP、議事録等） | プロジェクトの `docs/` | 案件ごとに新規作成 |
| **自社固有（単価、実績、サービス仕様等）** | **`arcadia/org-data/`** | **組織で維持・更新** |

---

## ディレクトリ構造

```
org-data/
  README.md                 # 本ファイル（セットアップ手順）
  rate-card.md              # 人月単価マスタ（ロール別・契約形態別）
  service-catalog.md        # 自社サービスカタログ（提案に引用可能な製品・サービス）
  company-profile.md        # 会社概要・導入実績・資格・認証
  whitepapers/
    index.md                # ホワイトペーパー索引（タグ・要約付き）
    *.md | *.pdf            # 個別のホワイトペーパー・技術資料
```

---

## セットアップ手順

### Step 1: テンプレートからファイルを作成

各 `.md` ファイルはテンプレート（コメント付き）として提供される。
自社の情報で埋めて完成させる。

```bash
# arcadia をクローン済みの前提
cd arcadia/org-data/

# 各ファイルを編集
# rate-card.md        → 自社の人月単価を記入
# service-catalog.md  → 提案に使えるサービス一覧を記入
# company-profile.md  → 会社概要・実績を記入
# whitepapers/index.md → ホワイトペーパーを登録
```

### Step 2: 機密情報の取り扱い

単価情報や原価率など、社外秘の情報を含む場合:

1. **Git 管理する場合**: プライベートリポジトリであることを確認
2. **Git 管理しない場合**: `.gitignore` に追加し、ローカルのみで管理

```gitignore
# 機密性の高い org-data を除外する場合
org-data/rate-card.md
org-data/whitepapers/*.pdf
```

### Step 3: プロジェクトへの適用

キックオフ時に Claude Code へ以下のように指示する:

```
arcadia/org-data/ の自社情報を読み込み、以下に反映して:
- 見積方針書の人月単価セクション → rate-card.md の標準単価を適用
- 提案戦略書の会社紹介・実績セクション → company-profile.md を参照
- 提案書の差別化セクション → service-catalog.md の該当サービスを引用
```

---

## 各ファイルの役割と参照先スキル

| ファイル | 内容 | 参照するスキル |
|---------|------|-------------|
| `rate-card.md` | 人月単価（ロール別・契約形態別）、原価率、値引基準 | **estimation-advisor** |
| `service-catalog.md` | 自社サービス・製品の仕様、特長、差別化ポイント | **proposal-writer** |
| `company-profile.md` | 会社概要、導入実績、資格・認証、パートナーシップ | **proposal-writer** |
| `whitepapers/index.md` | 技術ホワイトペーパーの索引（タグ・要約・引用ガイド） | **proposal-writer**, **estimation-advisor** |

---

## 更新サイクル

| 頻度 | 対象 | トリガー |
|------|------|---------|
| 四半期 | `rate-card.md` | 単価改定、原価率見直し |
| 随時 | `service-catalog.md` | 新サービスリリース、既存サービス仕様変更 |
| 半期 | `company-profile.md` | 実績追加、資格取得、組織変更 |
| 随時 | `whitepapers/` | 新規ホワイトペーパー発行 |

---

## 注意事項

- **単価情報は提案書本体には記載しない**（見積方針書・内部資料のみで使用）
- ホワイトペーパーから提案書に引用する場合は出典を明記する
- パートナー企業の情報は `company-profile.md` のパートナーセクションに記載する
- org-data の内容はプロジェクト横断で共有されるため、案件固有の情報は含めない
