# ARCADIA Platform Templates

プラットフォームアーキテクチャのテンプレート集。**クラウドインフラ**と**データ基盤**の2軸で構成される。

## 2軸のプラットフォーム選定

提案プロジェクトでは、以下の2つの軸でプラットフォームを選定する:

```
┌─────────────────────────────────────────────────┐
│                  platform/                       │
│                                                  │
│   ┌──────────────────┐  ┌────────────────────┐  │
│   │  cloud/           │  │  data/              │  │
│   │  クラウドインフラ  │  │  データ基盤         │  │
│   │                   │  │                     │  │
│   │  ├── aws/         │  │  ├── databricks/    │  │
│   │  ├── gcp/         │  │  ├── snowflake/     │  │
│   │  └── other/       │  │  ├── bigquery/      │  │
│   │                   │  │  └── common/        │  │
│   └──────────────────┘  └────────────────────┘  │
│                                                  │
│         {{CLOUD_PROVIDER}}  ×  {{PLATFORM_NAME}} │
└─────────────────────────────────────────────────┘
```

## 組み合わせパターン

| データ基盤 | AWS | GCP | Azure |
|-----------|-----|-----|-------|
| **Databricks** | cloud/aws + data/databricks | cloud/gcp + data/databricks | cloud/other + data/databricks |
| **Snowflake** | cloud/aws + data/snowflake | cloud/gcp + data/snowflake | cloud/other + data/snowflake |
| **BigQuery** | -- | cloud/gcp + data/bigquery | -- |

## テンプレート変数

| 変数 | 軸 | 説明 | 例 |
|------|-----|------|-----|
| `{{CLOUD_PROVIDER}}` | Cloud | クラウドプロバイダー | `AWS`, `GCP`, `Azure` |
| `{{PLATFORM_NAME}}` | Data | データ基盤名 | `Databricks`, `Snowflake`, `BigQuery` |
| `{{PLATFORM_TYPE}}` | Data | データ基盤カテゴリ | `Lakehouse`, `Cloud Data Warehouse` |

## 使い方

### 1. プロバイダー選定

プロジェクト初期化時（`guides/01-kickoff.md`）に2軸を選定:

```bash
# 例: AWS + Databricks の場合
# {{CLOUD_PROVIDER}} = AWS
# {{PLATFORM_NAME}} = Databricks
```

### 2. クラウドインフラ設計

```
platform/cloud/aws/ を参照してインフラ構成を設計して
```

### 3. データ基盤構築

```
platform/data/databricks/ のテンプレートを適用して
```
