# ARCADIA Cloud Platform Templates

クラウドインフラストラクチャの構成テンプレート集。VPC/ネットワーク、IAM/セキュリティ、ストレージ、監視の各領域について、プロバイダー別のベストプラクティスを提供する。

## プロバイダー対応表

| 機能カテゴリ | AWS | GCP | その他 (Azure等) |
|-------------|-----|-----|-----------------|
| **リージョン（国内）** | ap-northeast-1 (Tokyo) | asia-northeast1 (Tokyo) | プロバイダー依存 |
| **ネットワーク** | VPC / Subnet / Security Group | VPC / Subnet / Firewall Rules | プロバイダー依存 |
| **プライベート接続** | PrivateLink / VPC Endpoint | Private Service Connect | プロバイダー依存 |
| **オブジェクトストレージ** | S3 | Cloud Storage (GCS) | プロバイダー依存 |
| **IAM** | IAM Role / Policy | IAM Service Account / Role | プロバイダー依存 |
| **暗号化キー管理** | KMS | Cloud KMS | プロバイダー依存 |
| **監視** | CloudWatch | Cloud Monitoring | プロバイダー依存 |
| **ログ管理** | CloudTrail / CloudWatch Logs | Cloud Audit Logs / Cloud Logging | プロバイダー依存 |
| **IaC** | CloudFormation / Terraform | Deployment Manager / Terraform | Terraform 等 |

## 使い方

プロジェクト初期化時（`guides/01-kickoff.md`）に、`{{CLOUD_PROVIDER}}` テンプレート変数でクラウドプロバイダーを選択する。選択したプロバイダーのフォルダを参照し、インフラ構成の設計に活用する。

データ基盤（`platform/data/`）との組み合わせパターン:

| データ基盤 | 推奨クラウド | 備考 |
|-----------|------------|------|
| Databricks | AWS / GCP | マルチクラウド対応 |
| Snowflake | AWS / GCP | マルチクラウド対応 |
| BigQuery | GCP | GCP専用 |
