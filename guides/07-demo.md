## Claude Code Instructions

以下の指示文をClaude Codeにコピペして使用する。

### 1. デモアプリ仕様書の作成（初回のみ）

```
proposal-strategy.md とRFP要件を元に、デモアプリの仕様書を作成して。
以下の構成で:
1. コンセプト（ワンライナー）
2. 技術スタック
3. 画面一覧（各画面の概要、主要機能、モックデータ要件）
4. サンプルデータテーブル定義
5. 環境変数一覧
6. コーディング規約
出力先: demo-app-spec.md
```

### 2. demo-builderスキルによる画面追加

```
/demo-builder でダッシュボード画面を追加して（dashboardパターン）
```

```
/demo-builder で分析画面を追加して（chatパターン）
```

```
/demo-builder でシナリオ生成画面を追加して（wizardパターン）
```

### 3. モックデータの生成

```
demo-app-spec.md のサンプルデータテーブル定義を元に、モックデータを生成して。
クライアントのドメインに合ったリアリティのあるデータにして。
各テーブル50-100レコード程度。
```

### 4. 追加モックデータの生成

```
モックデータにキャンペーン実績データを追加して。
チャネル別（メール、プッシュ、SMS）のA/Bテスト結果を含めて。
```

### 5. データ基盤スクリプトの構築

```
platform/ ディレクトリにデータ基盤の構築スクリプトを作成して。
Medallionアーキテクチャ（Bronze→Silver→Gold）で:
00-setup: 環境設定
01-bronze: Raw データ取込
02-silver: クレンジング・変換
03-gold: 集計・分析用テーブル
言語: Python
```

### 6. プラットフォームAPI接続の実装

```
src/lib/platform-client.ts のAPI接続を実装して。
環境変数が未設定の場合はモックデータにフォールバックする設計で。
```

### 7. デモアプリのビルド確認

```
デモアプリをビルドして、エラーがないか確認して。
npm run build を実行して結果を報告して。
```

## Checklist

Phase完了の判定基準:

- [ ] `demo-app-spec.md` が存在し、全画面の仕様が定義されている
- [ ] 全画面（Ph1必須分）が実装され、ブラウザで表示可能である

- [ ] `npm run build` がエラーなく完了する
- [ ] `npm run dev` でローカル動作が確認できる
- [ ] データ基盤スクリプトが `platform/` に配置されている

- [ ] 人間によるUXレビューが完了し、主要フィードバックが反映されている
- [ ] プレゼン用デモシナリオに沿ったデモ走行が可能である

## Tips

- モックファースト開発を徹底する。プラットフォーム接続はすべての画面がモックで動作した後に着手する
- デモは「動くもの」を見せることが目的。完全な機能実装よりも、ストーリーに沿った画面遷移と視覚的インパクトを優先する
- サンプルデータはクライアントのドメインに合わせる（金融なら口座・取引・ローン等）
- Phase 5（提案書）にデモのスクリーンショットが必要な場合、早めに画面を完成させて連携する
- `demo-builder` スキルのパターン（dashboard / chat / wizard / journey / monitor）を活用すると、画面の初期構築が高速化する
- プレゼン日の前日にはデモ環境を凍結し、それ以降の変更は行わない
